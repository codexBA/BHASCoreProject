🐳 Docker Instrukcije za BHASCoreProject

📋 Pregled: Ovaj dokument objašnjava kako dockerizovati BHASCoreProject aplikaciju sa svim scenarijima povezivanja na bazu podataka.

📦 Korak 1: Dodavanje Docker Podrške u Visual Studio

1.1. Automatsko Kreiranje Dockerfile-a

  1. U Solution Explorer, desni klik na BHASCoreProject
  2. Odaberite: Add → Docker Support
  3. Izaberite: Linux (preporučeno) ili Windows
  4. Visual Studio će automatski kreirati Dockerfile
💡 Napomena: Visual Studio će kreirati optimizovan Dockerfile sa multi-stage build procesom.

🔧 Scenariji Povezivanja na Bazu Podataka

📍 SCENARIO 1: Lokalna Baza na Host Mašini (SQL Server na Windows-u)

1.1. Izmjena Connection String-a

U appsettings.json ili appsettings.Development.json:

{
  "ConnectionStrings": {
    "IdentityConnection": "Server=host.docker.internal,1433;Database=BHASIdentity;User Id=sa;Password=VašaLozinka;TrustServerCertificate=True;",
    "BusinessConnection": "Server=host.docker.internal,1433;Database=BHASBusiness;User Id=sa;Password=VašaLozinka;TrustServerCertificate=True;"
  },
  "AppSettings": {
    "Valuta": "EUR"
  }
}
⚠️ Važno: host.docker.internal omogućava Docker kontejneru da pristupa SQL Server-u koji radi na vašem Windows host računaru.

1.2. Pokretanje Docker Kontejnera

Opcija A - Iz Visual Studio:

  1. Pritisnite F5 ili kliknite Docker u toolbar-u
  2. Visual Studio automatski build-uje i pokreće kontejner

Opcija B - Ručno iz PowerShell-a:

# Navigirajte do root foldera projekta (gdje je .sln fajl)
cd X:\Codex\GitHub\Obuka\BHAS-PROJEKAT-CLASS\

# Build Docker image
docker build -t bhascoreproject:latest -f BHASCoreProject/Dockerfile .

# Pokrenite kontejner
docker run -d -p 8080:8080 -p 8081:8081 --name bhascoreapp bhascoreproject:latest

# Pristupite aplikaciji na:
# http://localhost:8080

1.3. Provjera i Upravljanje

# Provjerite da li kontejner radi
docker ps

# Pogledajte logove
docker logs bhascoreapp

# Zaustavite kontejner
docker stop bhascoreapp

# Pokrenite ponovo
docker start bhascoreapp

# Uklonite kontejner
docker rm bhascoreapp

🐳 SCENARIO 2: SQL Server u Docker Kontejneru

2.1. Kreiranje Docker Network-a

Docker network omogućava komunikaciju između kontejnera:

# Kreirajte custom bridge network
docker network create bhas-network

# Provjerite da je network kreiran
docker network ls
💡 Zašto Docker Network? Omogućava imenima kontejnera da funkcionišu kao DNS imena, tako da možete koristiti naziv kontejnera umjesto IP adrese.

2.2. Pokretanje SQL Server Kontejnera

# Pokrenite SQL Server 2022 kontejner na istom network-u
docker run -d `
  --name bhas-sqlserver `
  --network bhas-network `
  -e "ACCEPT_EULA=Y" `
  -e "SA_PASSWORD=VašaJakaSifra123!" `
  -p 1433:1433 `
  mcr.microsoft.com/mssql/server:2022-latest

# Provjerite da li SQL Server radi
docker logs bhas-sqlserver
⚠️ Sigurnost Lozinke: Lozinka mora sadržavati najmanje:
  • 8 karaktera
  • Velika i mala slova
  • Brojeve i specijalne karaktere

2.3. Kreiranje Baza Podataka (Opciono - ako ne koristite migracije)

# Pristupite SQL Server kontejneru
docker exec -it bhas-sqlserver /opt/mssql-tools/bin/sqlcmd `
  -S localhost -U sa -P "VašaJakaSifra123!"

# U sqlcmd prompt-u unesite:
CREATE DATABASE BHASIdentity;
GO
CREATE DATABASE BHASBusiness;
GO
EXIT
💡 Napomena: Vaš Program.cs automatski pokreće migracije pomoću authDbContext!.Database.Migrate();, tako da ovo možda nije potrebno.

2.4. Izmjena Connection String-a

Kreirajte appsettings.Docker.json u projektu:

{
  "ConnectionStrings": {
    "IdentityConnection": "Server=bhas-sqlserver,1433;Database=BHASIdentity;User Id=sa;Password=VašaJakaSifra123!;TrustServerCertificate=True;",
    "BusinessConnection": "Server=bhas-sqlserver,1433;Database=BHASBusiness;User Id=sa;Password=VašaJakaSifra123!;TrustServerCertificate=True;"
  },
  "AppSettings": {
    "Valuta": "EUR"
  }
}
⚠️ Ključna Razlika: Koristimo bhas-sqlserver (naziv kontejnera) umjesto host.docker.internal.

2.5. Izmjena Dockerfile-a da Koristi Docker Appsettings

U vašem Dockerfile, dodajte kopiranje fajla:

# Primjer multi-stage Dockerfile
FROM mcr.microsoft.com/dotnet/aspnet:10.0 AS base
WORKDIR /app
EXPOSE 8080
EXPOSE 8081

FROM mcr.microsoft.com/dotnet/sdk:10.0 AS build
WORKDIR /src
COPY ["BHASCoreProject/BHASCoreProject.csproj", "BHASCoreProject/"]
COPY ["BHASCore.Data/BHASCore.Data.csproj", "BHASCore.Data/"]
RUN dotnet restore "BHASCoreProject/BHASCoreProject.csproj"
COPY . .
WORKDIR "/src/BHASCoreProject"
RUN dotnet build "BHASCoreProject.csproj" -c Release -o /app/build

FROM build AS publish
RUN dotnet publish "BHASCoreProject.csproj" -c Release -o /app/publish /p:UseAppHost=false

FROM base AS final
WORKDIR /app
COPY --from=publish /app/publish .

# Kopirajte Docker-specifični appsettings (ako postoji)
COPY ["BHASCoreProject/appsettings.Docker.json", "./appsettings.Production.json"]

ENTRYPOINT ["dotnet", "BHASCoreProject.dll"]

2.6. Build i Pokretanje Web Aplikacije

# Build image
docker build -t bhascoreproject:latest -f BHASCoreProject/Dockerfile .

# Pokrenite kontejner na istom network-u
docker run -d `
  --name bhascoreapp `
  --network bhas-network `
  -p 8080:8080 `
  -p 8081:8081 `
  bhascoreproject:latest

# Provjerite da oba kontejnera rade na istom network-u
docker network inspect bhas-network
✅ Uspješno! Vaša aplikacija je pokrenuta i povezana sa SQL Server-om u odvojenom kontejneru putem bhas-network.

🚀 SCENARIO 3: Docker Compose (Preporučeno)

💡 Prednosti: Docker Compose omogućava pokretanje oba kontejnera (web app + SQL Server) jednom komandom, automatski kreira network i upravlja svim konfiguracijama.

3.1. Kreiranje Docker Fajlova (Ručno)

💡 Napomena za Visual Studio 2026: U Insiders verziji možda opcija "Container Orchestration Support" nije dostupna. Umjesto toga, kreirajte fajlove ručno - to vam daje više kontrole!

Već kreirani fajlovi u projektu:

  • BHASCoreProject/Dockerfile - Build instrukcije za web app
  • docker-compose.yml - Orchestracija servisa (root folder)
  • .dockerignore - Fajlovi koji se ne kopiraju u image

Alternativno - Ako želite dodati kroz Visual Studio:

  1. Možda trebate instalirati Docker Desktop i restartovati VS
  2. Ili: Add → Docker Support (kreiraće Dockerfile)
  3. Zatim ručno kreirajte docker-compose.yml

3.2. Kreiranje docker-compose.yml

Visual Studio će kreirati fajl, ali evo prilagođene verzije:

# =============================================================
# BHAS Core Project - Docker Compose Configuration
# =============================================================
# NAPOMENA: HTTPS je onemogućen za development (nema certifikata)
# Pristup: http://localhost:5100 (samo HTTP)
# =============================================================

services:
  # SQL Server Kontejner
  bhas-sqlserver:
    image: mcr.microsoft.com/mssql/server:2022-latest
    container_name: bhas-sqlserver
    environment:
      ACCEPT_EULA: "Y"
      SA_PASSWORD: "BhasAdmin123!"
      MSSQL_PID: "Developer"
    ports:
      - "14333:1433"  # Windows port 14333 -> Container port 1433
    volumes:
      - bhas-sqldata:/var/opt/mssql  # Perzistentno čuvanje SQL podataka
    networks:
      - bhas-network
    healthcheck:
      test: /opt/mssql-tools18/bin/sqlcmd -S localhost -U sa -P "BhasAdmin123!" -Q "SELECT 1" -C
      interval: 10s
      timeout: 5s
      retries: 10
      start_period: 30s
    restart: unless-stopped

  # Web Aplikacija
  bhascoreapp:
    image: bhascoreproject:latest
    container_name: bhascoreapp
    build:
      context: .
      dockerfile: BHASCoreProject/Dockerfile
    ports:
      - "5100:8080"  # HTTP samo (bez HTTPS)
    environment:
      ASPNETCORE_ENVIRONMENT: "Production"
      ASPNETCORE_URLS: "http://+:8080"
      ConnectionStrings__IdentityConnection: "Server=bhas-sqlserver,1433;Database=BHASIdentity;User Id=sa;Password=BhasAdmin123!;TrustServerCertificate=True;Encrypt=False;"
      ConnectionStrings__BusinessConnection: "Server=bhas-sqlserver,1433;Database=BHASBusiness;User Id=sa;Password=BhasAdmin123!;TrustServerCertificate=True;Encrypt=False;"
      AppSettings__Valuta: "EUR"
    depends_on:
      bhas-sqlserver:
        condition: service_healthy
    networks:
      - bhas-network
    restart: unless-stopped

volumes:
  bhas-sqldata:
    name: bhas-sqldata

networks:
  bhas-network:
    name: bhas-network
    driver: bridge
💡 Objašnjenje:
  • depends_on: Osigurava da SQL Server startuje prije web app-a
  • healthcheck: Provjerava da li je SQL Server spreman
  • volumes: Perzistentno čuvanje SQL podataka
  • environment: Override-uje appsettings preko environment varijabli
  • restart: unless-stopped: Automatski restart ako kontejner padne

3.3. Razumijevanje docker-compose.yml Strukture

YAML Format - Osnove:

  • Indentacija je KLJUČNA - koristite 2 spaces (ne TAB)
  • Key: Value - osnovni format
  • Liste - počinju sa -
  • Komentari - počinju sa #
# Glavne sekcije docker-compose.yml:

services:          # Kontejneri koje želimo pokrenuti
  service-name:    # Naziv servisa
    image:         # Docker image koji će se koristiti
    container_name:# Ime kontejnera
    environment:   # Environment varijable
    ports:         # Port mapping (HOST:CONTAINER)
    volumes:       # Disk storage mapping
    networks:      # Mreže za komunikaciju
    depends_on:    # Redoslijed pokretanja

volumes:           # Named volumes za perzistentne podatke
  volume-name:

networks:          # Custom networks
  network-name:

3.4. Docker Volumes - Gdje se Podaci Čuvaju?

Šta su Volumes?

Docker volumes su virtuelni hard disk-ovi koje Docker upravlja. Podaci ostaju NAKON što obrišete kontejnere!

U našem projektu:

volumes:
  bhas-sqldata:              # Ime volume-a
    name: bhas-sqldata       # Docker internal name

# Povezivanje sa SQL Server kontejnerom:
services:
  bhas-sqlserver:
    volumes:
      - bhas-sqldata:/var/opt/mssql
        ^^^^^^^^^^^^  ^^^^^^^^^^^^^^^
        |             |
        Volume name   Path UNUTAR kontejnera gdje se čuvaju SQL fajlovi

📂 Gdje se FIZIČKI čuvaju podaci na Windows-u?

# Docker Desktop čuva volumes na:
C:\Users\[VašeKorisničkoIme]\AppData\Local\Docker\wsl\data\

# Ili unutar WSL2 Linux distribuicije:
\\wsl$\docker-desktop-data\data\docker\volumes\bhas-sqldata\_data\

# Komanda za pregled:
docker volume inspect bhas-sqldata
⚠️ Važno: Ne pristupajte direktno tim fajlovima sa Windows-a! Koristite Docker komande za backup i restore.

3.5. Upravljanje Docker Volumes

# Pregled svih volumes
docker volume ls

# Detalji o volume-u
docker volume inspect bhas-sqldata

# Lokacija podataka
docker volume inspect bhas-sqldata --format '{{ .Mountpoint }}'

# Uklanjanje volume-a (BRIŠE PODATKE!)
docker volume rm bhas-sqldata

# Pravljenje backup-a SQL podataka
docker exec bhas-sqlserver /opt/mssql-tools18/bin/sqlcmd `
  -S localhost -U sa -P "BhasAdmin123!" `
  -Q "BACKUP DATABASE BHASIdentity TO DISK = '/var/opt/mssql/backup/BHASIdentity.bak'"

# Kopiraj backup na Windows
docker cp bhas-sqlserver:/var/opt/mssql/backup/BHASIdentity.bak ./backup/

3.6. Port Mapping - Detaljno Objašnjenje

Format: "HOST_PORT:CONTAINER_PORT"

ports:
  - "5100:8080"
     ^^^^  ^^^^
     |     └─── Port UNUTAR Docker kontejnera (ne mijenjate)
     └───────── Port na vašem Windows računaru (MOŽETE mijenjati)

Primjer:
  - "5100:8080"  # Pristup: http://localhost:5100
  - "9999:8080"  # Pristup: http://localhost:9999
  - "80:8080"    # Pristup: http://localhost (no port needed)
💡 Zašto različiti portovi?
  • 14333 (SQL) - Izbjegava konflikt sa lokalnim SQL Server-om na 1433
  • 5100 (Web) - Izbjegava konflikt sa IIS-om i drugim web serverima
  • Kontejneri međusobno komuniciraju preko INTERNOG network-a (ne koriste host portove)

3.7. Docker Networks - Komunikacija Između Kontejnera

# Kreiranje custom network-a
networks:
  bhas-network:
    name: bhas-network
    driver: bridge      # Najčešći tip - LAN unutar Docker-a

# Oba kontejnera na istom network-u
services:
  bhas-sqlserver:
    networks:
      - bhas-network  # Pridružen bhas-network-u

  bhascoreapp:
    networks:
      - bhas-network  # Takođe na bhas-network-u

🌐 Kako kontejneri komuniciraju?

# Web app koristi IME kontejnera kao DNS:
ConnectionStrings__IdentityConnection: "Server=bhas-sqlserver,1433;..."
                                                ^^^^^^^^^^^^^^
                                                Ime SQL kontejnera - radi kao hostname!

# Docker interno pretvara:
bhas-sqlserver → 172.18.0.2:1433 (internal IP)

# Web app NE koristi:
Server=localhost,14333  # ❌ Ovo je za Windows host
Server=localhost,1433   # ❌ Ovo je loopback u web kontejneru

# Web app KORISTI:
Server=bhas-sqlserver,1433  # ✅ DNS name preko Docker network-a

⚙️ Napredne Konfiguracije

HTTPS u Docker Kontejneru (Opciono - Naprednije)

💡 Zašto nema HTTPS u development?
  • Developer certifikati sa Windows-a nisu dostupni u Linux Docker kontejnerima
  • Za lokalni development, HTTP je potpuno dovoljan
  • Production deployment (Azure, AWS) automatski dodaje HTTPS

Ako ipak želite HTTPS za testiranje:

# 1. Generirajte self-signed certifikat
dotnet dev-certs https -ep ${HOME}/.aspnet/https/aspnetapp.pfx -p "YourPassword"
dotnet dev-certs https --trust

# 2. Kopirajte u projekat
mkdir https
copy ${HOME}\.aspnet\https\aspnetapp.pfx .\https\

# 3. Ažurirajte docker-compose.yml:
bhascoreapp:
  ports:
    - "5100:8080"   # HTTP
    - "5101:8081"   # HTTPS
  environment:
    ASPNETCORE_URLS: "https://+:8081;http://+:8080"
    ASPNETCORE_Kestrel__Certificates__Default__Path: /https/aspnetapp.pfx
    ASPNETCORE_Kestrel__Certificates__Default__Password: "YourPassword"
  volumes:
    - ./https:/https:ro  # Read-only mount
⚠️ Napomena: Self-signed certifikati će ipak pokazati upozorenje u browseru. Za pravu produkciju koristite Let's Encrypt ili Azure App Service certifikate.

Korištenje Environment Varijabli

Možete override-ovati bilo koju appsettings vrijednost kroz environment varijable:

docker run -d `
  -e "ConnectionStrings__IdentityConnection=Server=..." `
  -e "AppSettings__Valuta=BAM" `
  --name bhascoreapp `
  bhascoreproject:latest
💡 Format: Koristite __ (dva underscore-a) za navigaciju kroz JSON hijerarhiju.

Korištenje .env Fajla sa Docker Compose

Kreirajte .env fajl:

SA_PASSWORD=VašaJakaSifra123!
VALUTA=EUR

U docker-compose.yml:

environment:
  - SA_PASSWORD=${SA_PASSWORD}
  - AppSettings__Valuta=${VALUTA}

🔍 Rješavanje Problema (Troubleshooting)

Problem 1: Aplikacija ne može da se poveže na SQL Server

# Provjerite da li kontejneri mogu da ping-uju jedan drugi
docker exec bhascoreapp ping bhas-sqlserver

# Provjerite network
docker network inspect bhas-network

# Provjerite SQL Server logove
docker logs bhas-sqlserver

Problem 2: "Login failed for user 'sa'"

  • Provjerite da li je SQL Server kontejner potpuno pokrenut (pričekajte ~30 sekundi)
  • Provjerite da li je lozinka dovoljno jaka
  • Dodajte TrustServerCertificate=True; u connection string

Problem 3: Port 5100 ili 14333 je već zauzet

# Promijenite port mapiranje u docker-compose.yml
ports:
  - "9090:8080"      # Umjesto 5100
  - "15000:1433"     # Umjesto 14333

# Ili provjerite koji proces koristi port
netstat -ano | findstr :5100
netstat -ano | findstr :14333

# Zaustavite proces (sa PID-om iz output-a)
taskkill /PID  /F

Problem 4: Volume zauzima puno prostora

# Provjerite veličinu volume-a
docker system df -v

# Očistite stare SQL backup-e (ako ih pravite)
docker exec bhas-sqlserver rm -rf /var/opt/mssql/backup/*

# Kompletno čišćenje SQL podataka (OPREZ - briše sve!)
docker-compose down -v
docker volume rm bhas-sqldata
⚠️ UPOZORENJE: docker-compose down -v briše SVE podatke iz baze! Napravite backup prije nego izvršite ovu komandu.

Problem 5: "HTTPS endpoint error" greška

✅ Rješeno: HTTPS je onemogućen u trenutnoj konfiguraciji. Ako vidite ovu grešku:
  • Provjerite da imate ASPNETCORE_URLS: "http://+:8080" u docker-compose.yml
  • Provjerite da nema ASPNETCORE_HTTPS_PORTS environment varijable
  • Rebuild image: docker-compose up -d --build

🧹 Čišćenje Docker Resursa

Backup i Restore SQL Podataka

1. Backup SQL Baza (Unutar Kontejnera):

# Kreirajte backup folder na Windows-u
mkdir backup

# SQL Server backup komanda
docker exec bhas-sqlserver /opt/mssql-tools18/bin/sqlcmd `
  -S localhost -U sa -P "BhasAdmin123!" -C `
  -Q "BACKUP DATABASE BHASIdentity TO DISK = '/var/opt/mssql/data/BHASIdentity.bak' WITH FORMAT, INIT"

docker exec bhas-sqlserver /opt/mssql-tools18/bin/sqlcmd `
  -S localhost -U sa -P "BhasAdmin123!" -C `
  -Q "BACKUP DATABASE BHASBusiness TO DISK = '/var/opt/mssql/data/BHASBusiness.bak' WITH FORMAT, INIT"

# Kopirajte backup fajlove na Windows
docker cp bhas-sqlserver:/var/opt/mssql/data/BHASIdentity.bak ./backup/
docker cp bhas-sqlserver:/var/opt/mssql/data/BHASBusiness.bak ./backup/

2. Restore SQL Baza:

# Kopirajte backup fajlove u kontejner
docker cp ./backup/BHASIdentity.bak bhas-sqlserver:/var/opt/mssql/data/
docker cp ./backup/BHASBusiness.bak bhas-sqlserver:/var/opt/mssql/data/

# Restore baza
docker exec bhas-sqlserver /opt/mssql-tools18/bin/sqlcmd `
  -S localhost -U sa -P "BhasAdmin123!" -C `
  -Q "RESTORE DATABASE BHASIdentity FROM DISK = '/var/opt/mssql/data/BHASIdentity.bak' WITH REPLACE"

docker exec bhas-sqlserver /opt/mssql-tools18/bin/sqlcmd `
  -S localhost -U sa -P "BhasAdmin123!" -C `
  -Q "RESTORE DATABASE BHASBusiness FROM DISK = '/var/opt/mssql/data/BHASBusiness.bak' WITH REPLACE"

Čišćenje Resursa

# Zaustavite sve pokrenute kontejnere
docker stop $(docker ps -q)

# Uklonite sve zaustavljene kontejnere
docker container prune -f

# Uklonite nekorištene image-e
docker image prune -a -f

# Uklonite nekorištene volumes
docker volume prune -f

# Uklonite nekorištene networks
docker network prune -f

# Kompletan reset (OPREZ!)
docker system prune -a --volumes -f
⚠️ OPREZ: docker system prune -a --volumes briše SVE Docker resurse uključujući podatke u volumes!
